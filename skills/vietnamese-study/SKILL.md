# ベトナム語学習 音声教材作成スキル

このスキルは、ベトナム語の学習者が音声教材を作るための一連のワークフローを自動化する。
ユーザーは声調記号のないベトナム語文章を入力し、最終的にローカルにMP3ファイルが保存される。

## ワークフロー全体像

```
入力: 声調記号なしベトナム語 → 修正 → 単語解説 → Obsidian追記 → 5回繰り返しテキスト → ElevenLabs音声生成 → MP3ダウンロード
```

## Step 1: ベトナム語の修正と単語解説の作成

ユーザーから受け取った声調記号なしのベトナム語文章を、正確な声調記号付きベトナム語に修正する。

### 修正のポイント
- すべての母音に正しい声調記号（dấu）を付ける
- đ と d を正しく区別する
- 文脈から意図を推測する（例: "loan" → "loại", "loa" → "loại"）
- 明らかに不自然な文は自然な表現に修正する

### 単語解説のフォーマット
各文について、主要な単語・フレーズの日本語訳を付ける。既出の基本単語（anh, em, không, có...không など）は省略してよいが、新出語彙や重要な文法パターンは必ず説明する。

```
1. [修正後のベトナム語文]
    - [単語/フレーズ]: [日本語の意味]
    - [単語/フレーズ]: [日本語の意味]
```

## Step 2: Obsidianノートへの追記

### ファイルの場所を特定する
ユーザーの選択フォルダ内にある `ベトナム語会話.md` を探す。見つからない場合は `request_cowork_directory` でObsidian vaultフォルダへのアクセスを要求する。

### ファイル番号の自動判定
既存のファイル内容を読み込み、最後の `fileN` セクション番号を確認して、次の番号を自動で付与する。例えば最後が `file9` なら、新しいセクションは `file10` になる。

### 追記フォーマット
ファイルの末尾に以下の形式で追加する:

```
file[N]
1. [修正済みベトナム語文]
    - [単語]: [日本語の意味]
    - [単語]: [日本語の意味]

2. [修正済みベトナム語文]
    - [単語]: [日本語の意味]
...
```

余計な空白行や不要なフォーマットは入れない。既存のファイルのスタイルに合わせること。

## Step 3: 5回繰り返しテキストの作成

音声教材用に、各文章を5回ずつ繰り返すテキストを作成する。文と文の間は改行1つ、セクションとセクションの間は空行1つで区切る。

```
[文1]
[文1]
[文1]
[文1]
[文1]

[文2]
[文2]
[文2]
[文2]
[文2]

...
```

## Step 4: ElevenLabsで音声生成

### 前提条件
- Chrome in Chromeブラウザツールが必要
- ユーザーがElevenLabsにログイン済みであること

### 手順

1. `tabs_context_mcp` でタブ情報を取得する（`createIfEmpty: true`）
2. `https://elevenlabs.io/app/speech-synthesis/text-to-speech` にナビゲート
3. ログインが必要な場合、ユーザーに手動ログインを依頼して「続行して」を待つ
4. ページが読み込まれたら:
   - ポップアップやクッキーバナーがあれば閉じる（クッキーは「Reject All」を選択）
   - 紹介ポップアップがあれば「始める」で閉じる
5. メインテキストエリア（`read_page` で `filter: interactive` → テキストエリアのref）に繰り返しテキストを `form_input` で貼り付ける
6. モデルの確認: 「サポートされていない言語」の警告が出たら、「切り替え」をクリックしてEleven v3に変更する。v3の紹介ポップアップが出たら「始める」で閉じる
7. 「音声を生成」ボタンをクリック
8. 生成完了を待つ（10秒待ってからスクリーンショットで確認。プログレスバーが見えたら追加で待機）

### 注意点
- ElevenLabsのUIは頻繁に変わる可能性がある。要素が見つからない場合は `read_page` や `find` で再探索する
- ポップアップブロックでGoogleサインインが失敗することがある。その場合はユーザーに手動サインインを依頼する

## Step 5: MP3ダウンロードとリネーム

1. 「履歴」タブをクリックして、生成された音声を表示
2. 今日の日付の最新エントリを見つける
3. ダウンロードボタン（↓アイコン）をクリック → メニューから「MP3」を選択
   - メニューアイテムは `read_page` で `menuitem` として取得し、refを使ってクリックする（座標クリックだと反応しないことがある）
4. ダウンロード完了を待つ（5秒程度）
5. ユーザーのダウンロードフォルダへのアクセスを `request_cowork_directory` で要求（パス: `~/Downloads`）
6. ダウンロードフォルダ内の最新のElevenLabs MP3ファイルを特定する:
   ```bash
   ls -lt /path/to/Downloads/ | grep -i "elevenlabs.*mp3" | head -3
   ```
7. ファイルをObsidianのセクション番号に合わせてリネーム: `file[N].mp3`
8. リネームしたファイルをダウンロードフォルダに保存する

## 完了時のメッセージ

すべて完了したら、以下を報告する:
- Obsidianに追加したセクション番号（fileN）
- 修正した文章の一覧（修正前→修正後の対応）
- MP3ファイルのパスとファイル名
- Google Driveへの手動アップロードが必要な旨のリマインダー
